 Optionally, send an HTTP POST request to your server. Your server can response with a message to play to the caller in the HTTP response body.

We will send your web server a URL encoded form post with the following parameters.

Request content type application/x-www-form-urlencoded

Parameter Description
caller_number The incoming phone caller number.
called_number The tracking number that was dialed.
call_id The unique identifier for the incoming phone call.
input Any input captured from the DTMF codes from the incoming phone caller.

Your web server can respond with some control logic as well as conditionally tag a phone call.

The response from your web server should be a JSON object that contains a few controlling keys.

Response content type application/json

Key Definition
say Respond with a custom message.
next_key An integer key press value indicating the next menu key press to redirect the caller too.
tag Tag the call with the given tag.

An example response to conditionally route a call to a different menu option is below with the next_key option

{"next_key":2}

You may also add a custom message before routing to another menu item.

{"say":"Thank you for calling","next_key":2}

You can also tag the call dynamically from your servers.

{"tag":"SalesLead","next_key":2}
